## 🗄️ SAMPLE DATABASE FOR PRACTICE

### **Database Schema: Company Management System**

#### **1. departments**
```sql
CREATE TABLE departments (
    dept_id INT PRIMARY KEY AUTO_INCREMENT,
    dept_name VARCHAR(50) NOT NULL,
    location VARCHAR(50),
    budget DECIMAL(10,2)
);
```

#### **2. employees**
```sql
CREATE TABLE employees (
    emp_id INT PRIMARY KEY AUTO_INCREMENT,
    first_name VARCHAR(50) NOT NULL,
    last_name VARCHAR(50) NOT NULL,
    email VARCHAR(100) UNIQUE NOT NULL,
    salary DECIMAL(10,2) CHECK (salary > 0),
    hire_date DATE NOT NULL,
    dept_id INT,
    manager_id INT,
    FOREIGN KEY (dept_id) REFERENCES departments(dept_id),
    FOREIGN KEY (manager_id) REFERENCES employees(emp_id)
);
```

#### **3. projects**
```sql
CREATE TABLE projects (
    project_id INT PRIMARY KEY AUTO_INCREMENT,
    project_name VARCHAR(100) NOT NULL,
    start_date DATE,
    end_date DATE,
    budget DECIMAL(12,2)
);
```

#### **4. employee_projects** (Junction table for many-to-many relationship)
```sql
CREATE TABLE employee_projects (
    emp_id INT,
    project_id INT,
    role VARCHAR(50),
    assigned_date DATE,
    PRIMARY KEY (emp_id, project_id),
    FOREIGN KEY (emp_id) REFERENCES employees(emp_id),
    FOREIGN KEY (project_id) REFERENCES projects(project_id)
);
```

#### **5. customers**
```sql
CREATE TABLE customers (
    customer_id INT PRIMARY KEY AUTO_INCREMENT,
    customer_name VARCHAR(100) NOT NULL,
    email VARCHAR(100) UNIQUE,
    phone VARCHAR(20),
    city VARCHAR(50)
);
```

#### **6. orders**
```sql
CREATE TABLE orders (
    order_id INT PRIMARY KEY AUTO_INCREMENT,
    customer_id INT NOT NULL,
    order_date DATE NOT NULL,
    total_amount DECIMAL(10,2),
    status VARCHAR(20),
    FOREIGN KEY (customer_id) REFERENCES customers(customer_id)
);
```

#### **7. products**
```sql
CREATE TABLE products (
    product_id INT PRIMARY KEY AUTO_INCREMENT,
    product_name VARCHAR(100) NOT NULL,
    price DECIMAL(8,2) CHECK (price > 0),
    category VARCHAR(50),
    stock_quantity INT DEFAULT 0
);
```

#### **8. order_items**
```sql
CREATE TABLE order_items (
    order_id INT,
    product_id INT,
    quantity INT NOT NULL,
    unit_price DECIMAL(8,2),
    PRIMARY KEY (order_id, product_id),
    FOREIGN KEY (order_id) REFERENCES orders(order_id),
    FOREIGN KEY (product_id) REFERENCES products(product_id)
);
```

### **Table Relationships Explanation**

**Primary Keys (PK):**
- `departments.dept_id` - Uniquely identifies each department
- `employees.emp_id` - Uniquely identifies each employee
- `projects.project_id` - Uniquely identifies each project
- `customers.customer_id` - Uniquely identifies each customer
- `orders.order_id` - Uniquely identifies each order
- `products.product_id` - Uniquely identifies each product

**Foreign Keys (FK):**
- `employees.dept_id` → `departments.dept_id` (Many employees belong to one department)
- `employees.manager_id` → `employees.emp_id` (Self-referencing for hierarchical structure)
- `employee_projects.emp_id` → `employees.emp_id` (Many-to-many relationship)
- `employee_projects.project_id` → `projects.project_id` (Many-to-many relationship)
- `orders.customer_id` → `customers.customer_id` (Many orders belong to one customer)
- `order_items.order_id` → `orders.order_id` (One order has many items)
- `order_items.product_id` → `products.product_id` (Many orders can contain same product)

**Relationship Types:**
- **One-to-Many**: departments ↔ employees, customers ↔ orders, orders ↔ order_items
- **Many-to-Many**: employees ↔ projects (via junction table employee_projects)
- **Self-Referencing**: employees.manager_id (for organizational hierarchy)
